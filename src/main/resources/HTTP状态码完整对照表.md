# HTTP 状态码核心数据表

本表格仅保留生成 Java Record 类（含 code、name、message 字段）所需核心数据，字段对应关系：



* code：HTTP 状态码（数字类型）

* name：状态码英文名称（字符串类型）

* message：状态码描述（字符串类型）

## 1. 信息响应（1xx）



| code | name                | message                                                      |
| ---- | ------------------- | ------------------------------------------------------------ |
| 100  | Continue            | 服务器已接收请求头，客户端可继续发送请求体（常用于 POST 大文件场景） |
| 101  | Switching Protocols | 客户端请求切换协议（如 HTTP 切换到 WebSocket），服务器同意切换 |
| 102  | Processing          | 服务器正在处理请求，但尚未返回结果（主要用于 WebDAV 协议）   |
| 103  | Early Hints         | 服务器提前返回部分响应头（如`Link`），告知客户端后续可能的资源链接，加速页面加载 |

## 2. 成功响应（2xx）



| code | name                          | message                                                      |
| ---- | ----------------------------- | ------------------------------------------------------------ |
| 200  | OK                            | 请求成功，服务器返回对应资源（最常见成功状态码，如 GET 请求获取页面、接口返回数据） |
| 201  | Created                       | 请求已成功处理并创建新资源（如 POST 请求创建用户、上传文件后，返回新资源 URL） |
| 202  | Accepted                      | 请求已被接收，但尚未处理完成（常用于异步任务，如提交表单后后台排队处理） |
| 203  | Non-Authoritative Information | 服务器返回的资源信息来自第三方，而非原始服务器（内容正确但来源非权威） |
| 204  | No Content                    | 请求成功，但服务器无需返回响应体（如 DELETE 请求删除资源后，无需返回数据） |
| 205  | Reset Content                 | 请求成功，客户端需重置表单或页面（如提交表单后清空输入框）   |
| 206  | Partial Content               | 客户端发起范围请求（如断点续传），服务器返回部分资源（需配合`Range`请求头使用） |
| 207  | Multi-Status                  | 多状态响应（用于 WebDAV，返回多个资源的处理结果）            |
| 208  | Already Reported              | 已报告（用于 WebDAV，避免重复返回相同资源的处理状态）        |
| 226  | IM Used                       | 服务器已执行`GET`请求，且响应基于客户端提供的资源（如增量更新场景） |

## 3. 重定向（3xx）



| code | name               | message                                                      |
| ---- | ------------------ | ------------------------------------------------------------ |
| 300  | Multiple Choices   | 请求资源有多个选项（如不同格式的文件），客户端需选择其一     |
| 301  | Moved Permanently  | 资源已永久迁移到新 URL（浏览器会缓存该重定向，后续请求直接跳转新地址，SEO 权重转移） |
| 302  | Found              | 资源临时迁移到新 URL（浏览器不缓存，每次请求需重新获取重定向地址，SEO 权重不转移） |
| 303  | See Other          | 请求需通过`GET`方法访问新 URL（常用于 POST 请求后跳转，避免刷新页面重复提交） |
| 304  | Not Modified       | 资源未修改（客户端缓存的资源仍有效，服务器无需返回资源体，配合`If-Modified-Since`等缓存头使用） |
| 305  | Use Proxy          | 客户端需通过指定代理服务器访问资源（已废弃，因存在安全风险） |
| 306  | Switch Proxy       | 后续请求需使用指定代理（已废弃，无实际应用场景）             |
| 307  | Temporary Redirect | 临时重定向，保持原请求方法（如 POST 请求重定向后仍用 POST，区别于 302 可能转为 GET） |
| 308  | Permanent Redirect | 永久重定向，保持原请求方法（如 POST 请求重定向后仍用 POST，区别于 301 可能转为 GET，SEO 权重转移） |

## 4. 客户端错误（4xx）



| code | name                            | message                                                      |
| ---- | ------------------------------- | ------------------------------------------------------------ |
| 400  | Bad Request                     | 请求参数错误、格式无效或请求体语法错误（如 JSON 格式错误、必填参数缺失） |
| 401  | Unauthorized                    | 请求需身份验证（如未登录、Token 过期，服务器会返回`WWW-Authenticate`头提示认证方式） |
| 402  | Payment Required                | 预留状态码，用于需要付费才能访问资源的场景（目前极少实际应用） |
| 403  | Forbidden                       | 客户端已认证，但无权限访问资源（如普通用户访问管理员页面，服务器明确拒绝） |
| 404  | Not Found                       | 请求的资源不存在（如 URL 拼写错误、资源已删除，最常见的客户端错误码） |
| 405  | Method Not Allowed              | 请求方法不被允许（如用 POST 访问只支持 GET 的接口，服务器会返回`Allow`头说明允许的方法） |
| 406  | Not Acceptable                  | 服务器无法返回客户端可接受格式的资源（如客户端只接受 JSON，但服务器只能返回 XML） |
| 407  | Proxy Authentication Required   | 客户端需先通过代理服务器的身份验证（类似 401，但认证对象是代理） |
| 408  | Request Timeout                 | 客户端请求超时（服务器等待请求数据超过规定时间，如客户端发送请求过慢） |
| 409  | Conflict                        | 请求与服务器当前资源状态冲突（如修改已被他人更新的文件、创建重复的唯一标识资源） |
| 410  | Gone                            | 资源已永久删除，且无新 URL（区别于 404，明确告知资源不再存在，SEO 可移除该链接） |
| 411  | Length Required                 | 服务器要求请求包含`Content-Length`头，但客户端未提供（常用于 POST 请求） |
| 412  | Precondition Failed             | 请求头中的前置条件不满足（如`If-Match`验证失败，资源版本不匹配） |
| 413  | Payload Too Large               | 请求体过大（如上传文件超过服务器限制，服务器会返回`Retry-After`头建议重试时间） |
| 414  | URI Too Long                    | 请求 URL 过长（如 GET 请求参数过多，超出服务器处理上限）     |
| 415  | Unsupported Media Type          | 请求体格式不被支持（如服务器只接受 JSON，但客户端发送 FormData） |
| 416  | Range Not Satisfiable           | 客户端请求的资源范围无效（如请求文件的 1000-2000 字节，但文件仅 500 字节） |
| 417  | Expectation Failed              | 服务器无法满足`Expect`请求头的要求（如客户端要求`100-Continue`，但服务器不支持） |
| 418  | I'm a Teapot                    | 彩蛋状态码，标识服务器是茶壶，无法煮咖啡（源自愚人节玩笑，无实际业务用途） |
| 421  | Misdirected Request             | 请求被发送到无法处理该请求的服务器（如服务器未配置对应域名的解析） |
| 422  | Unprocessable Entity            | 请求格式正确，但语义错误（如提交表单时，字段值符合格式但不符合业务规则，如手机号格式正确但已被注册） |
| 423  | Locked                          | 请求资源被锁定（如 WebDAV 中文件正在被编辑，无法同时修改）   |
| 424  | Failed Dependency               | 请求依赖的其他请求失败（如批量操作中，前一个请求失败导致当前请求无法执行） |
| 425  | Too Early                       | 请求发送过早，服务器无法处理（如防止重复请求的场景，客户端需等待一段时间后重试） |
| 426  | Upgrade Required                | 客户端需升级协议才能访问资源（如 HTTP 升级到 HTTPS，服务器会返回`Upgrade`头说明需升级的协议） |
| 428  | Precondition Required           | 服务器要求请求包含前置条件（如`If-Modified-Since`，防止并发修改导致的数据不一致） |
| 429  | Too Many Requests               | 客户端请求过于频繁，超出服务器限流限制（服务器会返回`Retry-After`头建议重试时间） |
| 431  | Request Header Fields Too Large | 请求头过大（如 Cookie 过多、自定义头信息过长，服务器拒绝处理） |
| 451  | Unavailable For Legal Reasons   | 资源因法律原因无法访问（如违反版权、地区法规，服务器依法屏蔽） |

## 5. 服务器错误（5xx）



| code | name                            | message                                                      |
| ---- | ------------------------------- | ------------------------------------------------------------ |
| 500  | Internal Server Error           | 服务器内部未知错误（如代码 bug、数据库连接失败，最常见的服务器错误码） |
| 501  | Not Implemented                 | 服务器不支持请求的功能（如客户端使用服务器未实现的 HTTP 方法，如`PATCH`） |
| 502  | Bad Gateway                     | 网关或代理服务器收到上游服务器的无效响应（如反向代理后，后端服务宕机） |
| 503  | Service Unavailable             | 服务器暂时不可用（如服务器维护、负载过高，通常是临时状态，服务器会返回`Retry-After`头） |
| 504  | Gateway Timeout                 | 网关或代理服务器等待上游服务器响应超时（如后端服务处理请求过慢，超出网关超时时间） |
| 505  | HTTP Version Not Supported      | 服务器不支持客户端使用的 HTTP 版本（如客户端用 HTTP/3，但服务器只支持 HTTP/1.1） |
| 506  | Variant Also Negotiates         | 服务器内部配置错误，资源的协商机制出现循环（如不同格式的资源相互依赖协商） |
| 507  | Insufficient Storage            | 服务器存储空间不足，无法处理请求（如上传文件时服务器磁盘满） |
| 508  | Loop Detected                   | 服务器检测到请求处理陷入循环（如 WebDAV 中资源之间的依赖循环） |
| 510  | Not Extended                    | 服务器需要更多扩展信息才能处理请求（客户端需补充扩展头信息） |
| 511  | Network Authentication Required | 客户端需先通过网络认证（如公共 WiFi 的登录认证，未认证前无法访问互联网） |
